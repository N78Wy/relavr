## 背景
- 当前发送控制台把分辨率、帧率、码率固定展示为默认值，用户无法在开播前切换规格。
- Quest 实机打开当前 UI 时窗口默认较窄，控制台视觉上更接近手机纵向面板，不符合 VR 设备作为大屏窗口的使用预期。
- 现有 Compose 布局没有针对窄宽度与宽宽度做显式分级，窗口一旦偏窄，按钮和规格卡片容易被横向挤压，部分内容不可见。
- 页面里存在多处直接写死的深色文字，部分说明文字与深色背景对比度不足，影响可读性。

## 分阶段计划
1. 补齐任务文档与索引，梳理规格选项、Quest 默认窗口宽度和响应式布局策略。
2. 扩展 `StreamConfig` 规格候选与校验，保持前台服务透传链路一致。
3. 改造发送控制台 UI / ViewModel，支持规格选择、自适应布局和颜色对比度优化。
4. 补充单元测试、集成测试与 Compose UI 测试。
5. 执行根目录校验命令并回写结果。

## 关键决策
- 分辨率、帧率、码率采用独立选择器，不做预设档位。
- Quest 默认宽度同时通过 Manifest 自由窗口 `layout` 提示和 Compose 响应式布局双重兜底。
- 本轮规格候选固定为内置安全档位，不按设备能力做动态裁剪；如果个别设备无法稳定运行高档位，沿用现有错误路径。

## 实现结果
- `core/model` 为 `StreamConfig` 增加共享默认规格与候选列表，统一约束分辨率、帧率、码率只能从内置档位中选择，并让前台服务反序列化复用同一套默认值。
- `feature/stream-control` 将“默认规格”只读展示改为可选的“视频规格”卡片，新增分辨率、帧率、码率独立选择器；布局按宽度切换为紧凑单列、居中单列和宽屏双列，窄宽度下开始/停止按钮改为纵向堆叠，避免内容被横向挤出。
- `app` 为 `MainActivity` 增加自由窗口默认宽度、默认高度和最小宽度提示，让 Quest 打开时更接近平板窗口；同时扩展主题色板和控件颜色，提升深色背景上的文字与边框对比度。
- 单元测试、集成测试与 Compose UI 测试已同步覆盖规格校验、规格透传和窄宽度场景可见性。
